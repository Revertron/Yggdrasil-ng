# Interop Integration Tests

Automated interop tests between **yggdrasil-go** (Go reference) and
**yggdrasil-ng** (Rust reimplementation). Verifies that both implementations
can peer over all supported transports.

## Test Matrix

18 tests total — each transport is tested in both directions (Go server / Rust
client and vice-versa) across three address types:

| Transport | IPv4 (`127.0.0.1`) | IPv6 (`[::1]`) | Hostname (`localhost`) |
|-----------|:--:|:--:|:--:|
| **TLS**   | Go->Rust, Rust->Go | Go->Rust, Rust->Go | Go->Rust, Rust->Go |
| **QUIC**  | Go->Rust, Rust->Go | Go->Rust, Rust->Go | Go->Rust, Rust->Go |
| **WS**    | Go->Rust, Rust->Go | Go->Rust, Rust->Go | Go->Rust, Rust->Go |

## Prerequisites

1. **Build yggdrasil-ng** with all transports enabled:

   ```bash
   cd /path/to/Yggdrasil-ng
   cargo build --features all-transports
   ```

2. **Build yggdrasil-go**:

   ```bash
   cd /path/to/yggdrasil-go
   go build -o yggdrasil ./cmd/yggdrasil
   ```

3. **Python 3.8+** — only stdlib modules are used, no pip packages needed.

## Usage

Set `YGGDRASIL_GO_DIR` to the directory containing the Go `yggdrasil` binary,
then run the script:

```bash
export YGGDRASIL_GO_DIR=/path/to/yggdrasil-go
python3 contrib/mixnet_test/interop_test.py
```

The Rust binary is found automatically at `../../target/debug/yggdrasil`
relative to the script.

### Options

```
  -v, --verbose         Show node stdout/stderr (useful for debugging)
  -f, --filter PATTERN  Only run tests whose name contains PATTERN
  -t, --timeout SECS    Peer connection timeout in seconds (default: 15)
```

### Examples

```bash
# Run all tests
export YGGDRASIL_GO_DIR=../yggdrasil-go
python3 contrib/mixnet_test/interop_test.py

# Run only QUIC tests with verbose output
python3 contrib/mixnet_test/interop_test.py -v -f quic

# Run only IPv6 tests
python3 contrib/mixnet_test/interop_test.py -f ipv6

# Run only TLS tests with longer timeout
python3 contrib/mixnet_test/interop_test.py -f tls -t 30

# Run only localhost tests
python3 contrib/mixnet_test/interop_test.py -f localhost
```

## How It Works

- Both daemons run with `IfName = "none"` — **no TUN interface, no root
  required**.
- Connectivity is verified via the admin API (`getPeers`). A successful
  `getPeers` response with `up: true` proves:
  1. Transport-level connection (TCP/TLS/QUIC/WS)
  2. Yggdrasil metalink handshake (version check, key exchange, signature)
  3. Ironwood peer registration
- Configs are generated by calling each binary's `--genconf` and patching
  listen/peer/admin fields.
- Each test gets unique ports (listen: 19001+, admin: 19101+) to avoid
  conflicts.
- Processes are managed with `os.setsid` + `os.killpg` for clean shutdown.
